# 像素风机甲对战游戏 - 设计规格

## 1. 为什么做这个游戏

创造一个复古 16-bit 风格的机甲对战游戏，支持双人本地对战和 P2P 网络联机。玩家操控机甲进行近身格斗，体验紧张刺激的对战快感。

## 2. 核心玩法

### 战斗系统
- **战斗类型**：近身格斗
- **血量**：100 HP/机甲
- **普通攻击**：造成 15 点伤害
- **防御状态**：减免 50% 伤害
- **攻击冷却**：500ms
- **胜负条件**：对方 HP 归零即获胜

### 操控方式
| 玩家 | 角色 | W/↑ | A/← | D/→ | J/. | K// |
|------|------|-----|-----|-----|-----|-----|
| 玩家1 | 蓝色机甲 | 跳跃 | 左移 | 右移 | 攻击 | 防御 |
| 玩家2 | 红色机甲 | 跳跃 | 左移 | 右移 | 攻击 | 防御 |

### 游戏模式
1. **本地双人对战**：两人共用键盘对战
2. **P2P 网络对战**：通过 PeerJS 建立直连

## 3. 技术实现

### 技术栈
- **单 HTML 文件**：所有代码内嵌，无需构建工具
- **渲染**：Canvas 2D API
- **P2P 通信**：PeerJS（WebRTC 封装）
- **帧率**：60 FPS

### 代码架构
```
单文件结构：
├── HTML/CSS 样式
├── PeerJS P2P 模块
│   ├── 连接管理（房间 ID）
│   ├── 消息同步（操作指令）
│   └── 状态同步（血量）
├── 游戏引擎
│   ├── 主循环（requestAnimationFrame）
│   ├── 输入处理（键盘事件）
│   ├── 物理系统（重力、碰撞）
│   └── 碰撞检测（攻击判定）
├── 角色系统
│   ├── 机甲类（Mech）
│   ├── 动画状态机
│   └── 动作（待机、移动、跳跃、攻击、防御、受击）
├── 像素渲染
│   ├── 背景绘制（战斗平台）
│   ├── 机甲绘制（Canvas 像素绘制）
│   ├── 特效绘制（受击闪烁）
│   └── UI 绘制（血条、状态）
└── UI 界面
    ├── 菜单系统
    ├── 连接界面
    └── 游戏结束画面
```

## 4. 视觉设计

### 像素艺术规格
- **机甲尺寸**：64x64 像素
- **缩放比例**：3x（画布显示 192x192）
- **动画帧率**：8 FPS

### 机甲配色
**蓝色机甲（玩家1）**
- 主色：#1a4f8a（深蓝）
- 辅色：#5c7c99（银灰）
- 高光：#f5d742（黄色）
- 装甲：#2d6bb3（中蓝）

**红色机甲（玩家2）**
- 主色：#8a2d2d（深红）
- 辅色：#4a4a4a（黑灰）
- 高光：#4fc3f7（青色）
- 装甲：#c94444（亮红）

### 场景设计
- **背景**：深色工业风，渐变 #1a1a2e → #16213e
- **战斗平台**：灰色金属质感，#4a4a4a 底色 + #6a6a6a 高光
- **边缘处理**：平台两侧有警示条纹

### 动画帧
每个动作 4-6 帧序列：
- **待机**：微妙的上下浮动（呼吸感）
- **行走**：腿部交替，左右倾斜
- **跳跃**：上升蓄力 → 空中 → 下落
- **攻击**：前冲挥拳，拳臂前伸
- **防御**：机甲前倾，举起手臂护盾
- **受击**：闪烁 3 次，无敌帧

## 5. P2P 联机实现

### 连接流程
1. 房主点击"创建房间"，生成唯一房间 ID
2. 房客输入房间 ID，点击"加入房间"
3. PeerJS 通过信令服务器建立直连
4. 连接成功后同步初始状态

### 数据同步
- **操作指令**：移动方向、跳跃、攻击、防御（实时发送）
- **游戏状态**：血量（攻击后同步）
- **同步频率**：输入指令实时同步，状态变化时同步

## 6. 界面流程

```
[主菜单]
    ├── 本地对战 → [加载游戏] → [对战画面]
    └── 网络对战
            ├── 创建房间 → [等待连接] → [对战画面]
            └── 加入房间 → [连接中...] → [对战画面]
```
