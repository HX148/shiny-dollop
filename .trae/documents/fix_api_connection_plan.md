# 修复 API 连接问题计划

## 问题分析

根据浏览器控制台的错误日志，主要问题是：
- 前端无法连接到后端 API 服务器
- 错误信息：`net::ERR_ABORTED` 和 `Failed to fetch news`
- 当前只启动了前端开发服务器，后端服务没有运行

## 根因

1. 项目配置了前后端分离架构：
   - 前端运行在 Vite 开发服务器（默认端口 5173）
   - 后端 Express API 应运行在端口 3001
   - Vite 配置了代理将 `/api` 请求转发到 `http://localhost:3001`

2. 当前开发环境没有同时启动前端和后端

## 修复方案

### 1. 确保后端服务器能正确启动
- 检查并修复可能的 TypeScript 编译问题
- 确保后端 API 路由能正常响应

### 2. 改进开发体验
- 提供清晰的启动说明
- 确保 `npm run dev` 能同时启动前后端

### 3. 增强错误处理
- 改进前端的错误提示信息
- 确保在 API 不可用时显示友好的用户界面

## 需要修改的文件

1. `README.md` - 添加完整的启动说明
2. 可能需要检查 TypeScript 编译配置

## 执行步骤

1. 首先尝试启动后端服务，检查是否有编译或运行时错误
2. 确保前后端能同时正常运行
3. 更新文档说明
4. 验证修复效果

## 预期结果

- 用户运行 `npm run dev` 后，前端和后端同时启动
- API 请求正常工作
- 新闻数据能正常显示
